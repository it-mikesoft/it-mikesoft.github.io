# Musica Podcast — Prompt per Suno.com

Sottofondo per episodi da ~20 minuti. Il podcast è un racconto pacato,
conversazionale, per un pubblico che ascolta in auto o camminando. La musica
deve **accompagnare**, mai sovrastare o creare urgenza. Niente techno, niente
cyberpunk, niente "breaking news". Pensa a: tarda sera, studio caldo, libro
aperto, un amico che racconta.

## Regole generali

1. **Più varianti per ogni mood** — lo script ruota `bg-tech.mp3`, `bg-tech1.mp3`, ecc.
2. Ogni traccia almeno **2 minuti** in Custom Mode con sezioni interne (build, breathing,
   ritorno) — così anche in loop non suona piatto.
3. Tutte **strumentali**, nessuna voce, nessun vocalizzo.
4. Volume in produzione: -22dB sotto la voce (`musicVolume: 0.08`).

## Workflow Suno

- **Custom Mode** → "Instrumental" ON.
- **Style**: copia il blocco STYLE qui sotto.
- **Lyrics/Description**: copia il blocco STRUCTURE.
- Genera 2-3 varianti per mood, scegli la più calda e meno invadente.

## Note pratiche

- **Lunghezza minima**: 2 minuti. Tracce più lunghe = meno loop udibili.
- **Volume**: in `data/podcast-config.json` regola `musicVolume` (default 0.08 = -22dB sotto la voce).
- **Quanti file generare**: 2-3 varianti per ogni mood bastano.
- **Test rapido**: `npm run podcast` e ascolta.
- **Mood mapping**: lo script assegna i mood ai capitoli via `[CHAPTER mood=tech|geo|soft|upbeat]`.
- **Naming**: `bg-tech.mp3`, `bg-tech1.mp3`, `bg-tech2.mp3`, ... — il file senza numero è la variante "0".
